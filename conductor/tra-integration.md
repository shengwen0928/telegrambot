# 台鐵會員搶票整合計畫

## 1. 目標
將台灣鐵路（TRA）訂票系統整合至現有的 LINE Bot 框架中，支援會員登入、全卡片式站點選擇、日期設定與自動搶票監控，同時確保與現有和欣客運功能互不干擾。

## 2. 關鍵檔案與變更
- `src/tr_api.py`: 實作台鐵 API 通訊，包含 CSRF Token 獲取、會員登入模擬（含驗證碼處理預備）及車次查詢。
- `src/tr_monitor.py`: 實作台鐵專屬監控循環，包含定期查票、剩餘座位判斷與自動執行 `bookingTicket` POST 請求。
- `src/tr_stations.py`: 提供台鐵全台主要車站代碼映射表。
- `line_bot.py`: 
    - 整合業者選擇卡片（和欣 vs 台鐵）。
    - 實作台鐵身分證（User ID）與會員密碼的輸入、驗證與 `users.json` 儲存邏輯。
    - 實作台鐵車站輪播選擇器（使用 `tr_stations.py` 資料）。
    - 升級 `start_monitor_task` 與 `LineNotifier` 以支援雙業者任務啟動與精緻通知卡片。
    - **徹底清理**檔案中因先前重構產生的重複函式與邏輯衝突。

## 3. 實作步驟
1. **API 強化**: 在 `src/tr_api.py` 中完善登入後的 Cookie 維持機制。
2. **監控邏輯**: 在 `src/tr_monitor.py` 中寫入具體的「找位子 -> 填表 -> 送出」代碼。
3. **UI 整合與清理**:
    - 使用 `generalist` 子智能體對 `line_bot.py` 進行一次性的大規模清理與代碼整合。
    - 確保 `handle_message` 中的狀態機（States）能正確引導使用者進入台鐵流程。
    - 確保所有通知（啟動、成功、錯誤）均統一為 Mega Bubble 尺寸的 Flex Message。
4. **驗證**: 在 GCE 環境啟動 `uvicorn` 進行全流程測試。

## 4. 驗證與測試
- 測試「客運:台鐵」按鈕是否能觸發身分證輸入。
- 測試台鐵車站選單是否能正確顯示並選擇（如：1000-臺北）。
- 測試搶票任務啟動後，是否回傳精緻的「任務收執聯」卡片。
- 確認和欣客運功能是否依然運作正常，且帳密儲存獨立。

## 5. 提交與部署
- 完成代碼後進行 `git commit`。
- `git push` 到 GitHub 以便 GCE 端同步。
- 重啟 `uvicorn` 服務。
