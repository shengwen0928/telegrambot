# 和欣客運全自動監控搶票系統 - 設計規格書 (Spec)

## 1. 專案目標
開發一個基於 Python 的自動化工具，監控和欣客運官網餘票，並在發現符合條件的班次時，自動完成訂位動作並透過 Telegram 通知使用者。

## 2. 使用者需求
- **全自動化：** 從登入到搶票完全無須人工干預。
- **極速反應：** 使用 API (Requests) 級別的操作，避開瀏覽器渲染的延遲。
- **AI 辨識：** 使用 `ddddocr` 本地辨識 4 位數圖形驗證碼，實現自動登入。
- **智慧監控：** 設定特定日期與時段，以 1-3 秒隨機間隔輪詢。
- **即時通知：** 訂位成功後立即透過 Telegram Bot 通知。

## 3. 技術棧
- **語言：** Python 3.10+
- **網路請求：** `httpx` (支援非同步，效能較佳)
- **驗證碼辨識：** `ddddocr`
- **環境變數管理：** `python-dotenv`
- **通知介面：** Telegram Bot API

## 4. 系統模組設計

### A. 認證模組 (Auth Module)
- **登入流程：**
    1. 請求登入頁面，獲取 Session 與驗證碼圖片 URL。
    2. 下載驗證碼圖片並傳遞給 `ddddocr`。
    3. 發送登入 POST 請求（含手機號碼、密碼與 OCR 結果）。
    4. **異常處理：** 若驗證碼辨識錯誤導致登入失敗，自動重試直至成功。

### B. 監控引擎 (Monitor Engine)
- **輸入參數：** 日期、起點站 ID、終點站 ID、目標時段（例如 17:00 - 22:00）。
- **邏輯：** 
    - 循環呼叫「查詢班次」API。
    - 比對 API 回傳的班次資料，檢查是否有票。
    - 採用 `random.uniform(1.0, 3.0)` 作為請求間隔。

### C. 訂位模組 (Booking Module)
- **觸發條件：** 監控引擎發現目標班次有票。
- **執行動作：**
    1. 立即選擇該班次。
    2. 發送確認訂位請求。
    3. 獲取訂單編號或交易序號。

### D. 通知模組 (Notification Module)
- 使用 Telegram Bot API。
- **發送時機：** 程式啟動、登入成功、訂位成功、發生致命錯誤。
